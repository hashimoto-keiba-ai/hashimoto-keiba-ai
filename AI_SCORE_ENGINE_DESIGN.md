# 橋本競馬AI Phase2-2 AI指数自動計算エンジン設計

## 目的

CSVで取り込んだ出馬表データから、AI指数・神穴指数・危険人気馬指数を自動計算できるようにするための設計を整理します。

Phase2-2では、既存のブラウザ版、CSV取込、localStorage保存、AI指数ランキング、神穴ランキング、危険人気馬ランキング、三連単生成の導線を壊さず、計算ロジックを段階的に接続します。

## 前提方針

- 入力元はPhase2-1で定義した出馬表CSV、オッズCSV、手入力補完データを想定します。
- 不足データがある場合は、該当項目を0点または暫定補正として扱い、根拠メモに「データ不足」を残します。
- 競馬場別補正、距離別補正、馬場状態、展開予測、人気/オッズを分離し、後から重みを調整できる構造にします。
- 最終的な各指数はランキング表示、買い目生成、結果検証、自己進化AIの重み調整へ連動させます。

## 1. AI指数の構成要素

AI指数は、出走馬の総合的な勝ち切り能力と好走期待値を表す基礎指数です。以下の構成要素を加算して算出します。

| 構成要素 | 評価内容 | 主な参照データ |
| --- | --- | --- |
| 能力指数 | 近走成績、着差、走破時計、上がり、クラス実績を評価する | 過去走、クラス、着順、タイム、上がり |
| 位置指数 | 想定位置取り、先行力、追走力、枠順からのポジション利を評価する | 脚質、枠番、馬番、ペース予測 |
| 展開一致度 | レース全体の想定ペースと各馬の脚質が合うかを評価する | 逃げ/先行/差し/追込、出走メンバー構成 |
| 騎手補正 | 騎手のコース実績、距離実績、人気別成績、乗替効果を評価する | 騎手DB、騎手名、乗替情報 |
| 調教師補正 | 厩舎の仕上げ力、コース傾向、休み明け成績、勝負度を評価する | 調教師DB、厩舎名、ローテ |
| 血統補正 | コース、距離、馬場、季節に対する血統適性を評価する | 父、母父、血統DB |
| 調教補正 | 追い切り内容、時計、併せ馬、上昇度、最終追いの評価を反映する | 調教評価、調教時計、調教メモ |
| 馬場適性 | 芝/ダート、良/稍重/重/不良、馬場バイアスへの適性を評価する | 馬場状態、過去走馬場、馬場バイアスAI |
| 距離適性 | 今回距離と過去好走距離の一致度を評価する | 距離、過去走距離、血統距離傾向 |
| 人気期待値 | 人気やオッズに対して指数上の妙味があるかを評価する | 人気、単勝オッズ、複勝レンジ |

## 2. 基本計算式

AI指数は、各構成要素を加算して算出します。

```text
AI指数 =
能力指数
+ 位置指数
+ 展開一致度
+ 騎手補正
+ 調教師補正
+ 血統補正
+ 調教補正
+ 馬場適性
+ 距離適性
+ 人気期待値
```

### 正規化方針

- 各項目は初期実装では原則 `-10` 〜 `+20` 程度の補正幅に収めます。
- 能力指数は中心項目のため、他項目より大きい配点にできます。
- 競馬場別・距離別補正は、能力指数そのものではなく、位置指数、展開一致度、馬場適性、距離適性に上乗せします。
- 最終AI指数はランキング表示しやすいよう、必要に応じて `0` 〜 `100` へスケーリングします。

### 出力項目

| 出力項目 | 内容 |
| --- | --- |
| aiScore | 最終AI指数 |
| aiRank | AI指数順位 |
| aiScoreBreakdown | 各構成要素の内訳 |
| aiReason | 上位評価・減点理由のメモ |
| provisionalFlags | データ不足や暫定評価のフラグ |

## 3. 神穴指数

神穴指数は、人気薄の中で配当妙味と好走条件が重なっている馬を抽出する指数です。

```text
神穴指数 =
人気妙味
+ 展開ハマり度
+ 枠順脚質一致
+ 騎手穴補正
+ 調教上昇
+ 万馬券パターン一致
```

### 構成要素

| 構成要素 | 評価内容 |
| --- | --- |
| 人気妙味 | 人気・オッズが低評価でも、AI指数や適性が高い馬を加点する |
| 展開ハマり度 | 逃げ残り、前崩れ差し込み、内前有利など、展開がハマる馬を加点する |
| 枠順脚質一致 | コース別に有利な枠順と脚質が一致する場合に加点する |
| 騎手穴補正 | 人気薄での好走率、ローカル巧者、乗替妙味を評価する |
| 調教上昇 | 前走からの上積み、攻め強化、状態上昇がある馬を評価する |
| 万馬券パターン一致 | 万馬券DB・神レースDBに蓄積した高配当再現条件と一致する馬を加点する |

### 判定ランク

| ランク | 目安 | 用途 |
| --- | --- | --- |
| S | 神穴指数が非常に高く、複数条件が一致 | 三連単の2・3着厚め、万馬券型の軸候補 |
| A | 配当妙味と展開利が明確 | 本線の相手、攻撃型の候補 |
| B | 一部条件が一致 | 押さえ穴候補 |
| C | 妙味不足または根拠不足 | 原則軽視 |

## 4. 危険人気馬指数

危険人気馬指数は、人気上位馬の中から「人気ほど信頼しにくい馬」を抽出する指数です。

```text
危険人気馬指数 =
過剰人気
+ 展開不一致
+ 位置不利
+ 馬場不一致
+ 距離不安
+ 調教不安
+ 斤量/馬体重不安
```

### 構成要素

| 構成要素 | 評価内容 |
| --- | --- |
| 過剰人気 | オッズや人気がAI指数、近走内容、適性に対して売れすぎている馬を加点する |
| 展開不一致 | 想定ペースや隊列が脚質に合わない馬を加点する |
| 位置不利 | 枠順、馬番、出遅れ癖、包まれリスクなど位置取りの不安を加点する |
| 馬場不一致 | 今回馬場と過去好走馬場が合わない馬を加点する |
| 距離不安 | 距離延長/短縮、血統傾向、過去実績から不安がある馬を加点する |
| 調教不安 | 追い切り低調、併せ遅れ、急仕上げ、状態下降を加点する |
| 斤量/馬体重不安 | 斤量増、馬体重大幅増減、成長途上/衰えのリスクを加点する |

### 判定ランク

| ランク | 目安 | 買い目への扱い |
| --- | --- | --- |
| S | 危険人気馬指数が非常に高い | 本線から除外または大幅軽視 |
| A | 明確な不安材料が複数ある | 1着固定を避け、相手までに制限 |
| B | 一部不安がある | 点数調整時に優先度を下げる |
| C | 危険材料が少ない | 通常評価 |

## 5. 競馬場別補正

競馬場別補正は、コース形態、直線長、坂、コーナー数、ローカル適性、枠順傾向を反映します。

| 競馬場 | 補正方針 |
| --- | --- |
| 中山 | 小回り、急坂、器用さ、先行力、内枠適性を重視する。差し馬は展開利が明確な場合に加点する。 |
| 東京 | 長い直線、瞬発力、上がり性能、差し脚、広いコース適性を重視する。先行馬は持続力が必要。 |
| 京都 | 平坦適性、下り坂加速、立ち回り、内回り/外回り差を重視する。長距離では折り合いを加点する。 |
| 阪神 | 急坂、持続力、パワー、内回りの機動力、外回りの瞬発力を分けて評価する。 |
| 中京 | 坂、長い直線、左回り適性、差し届き傾向を評価する。ダートは先行力と持続力を重視する。 |
| 新潟 | 外回りの直線スピード、内回りの先行力、直線競馬の枠順適性を分けて補正する。 |
| 福島 | 小回り、先行力、ローカル適性、コーナー加速、馬場悪化耐性を重視する。 |
| 小倉 | 小回り、スピード持続、平坦適性、開幕週の先行力、開催後半の差し適性を補正する。 |

## 6. 距離別補正

距離別補正は、芝/ダート、短距離/マイル/中距離/長距離、ペース傾向、脚質適性を反映します。

| 距離カテゴリ | 補正方針 |
| --- | --- |
| 芝短距離 | 先行力、テンの速さ、枠順、スピード持続力を重視する。差し馬はハイペース想定で加点する。 |
| 芝マイル | 瞬発力、位置取り、折り合い、直線性能をバランス評価する。コース別の直線長を反映する。 |
| 芝中距離 | 基礎能力、持続力、瞬発力、騎手のペース判断、距離適性を重視する。 |
| 芝長距離 | スタミナ、折り合い、長距離血統、騎手の仕掛け、斤量耐性を重視する。 |
| ダート短距離 | 先行力、テンの速さ、砂適性、枠順、馬格を重視する。出遅れリスクは大きく減点する。 |
| ダート1400 | 先行力と差し脚のバランス、距離短縮/延長適性、外枠適性を評価する。 |
| ダート1800 | 先行力、コーナー4回の立ち回り、持続力、パワー、騎手の位置取りを重視する。 |
| ダート中長距離 | スタミナ、先行持続力、揉まれ耐性、斤量、ローテーションを重視する。 |

## 7. Phase2-2 実装順

| Step | 実装内容 | 完了目安 |
| --- | --- | --- |
| Step1 | 計算ロジック定義 | AI指数、神穴指数、危険人気馬指数の構成要素、初期重み、出力項目を定義する |
| Step2 | CSVデータとの接続 | 出馬表CSV、オッズCSV、手入力補完データを各指数の入力項目へマッピングする |
| Step3 | AI指数ランキング自動更新 | CSV取込後にAI指数を再計算し、AI指数ランキングへ反映する |
| Step4 | 神穴ランキング自動更新 | 人気妙味、展開ハマり度、万馬券パターン一致を計算し、神穴ランキングへ反映する |
| Step5 | 危険人気馬ランキング自動更新 | 人気上位馬を対象に危険人気馬指数を計算し、危険人気馬ランキングへ反映する |
| Step6 | 三連単生成へ接続 | AI指数上位、神穴候補、危険人気馬軽視ルールを三連単自動生成へ渡す |

## データ接続イメージ

```text
出馬表CSV / オッズCSV / 手入力補完
  ↓
CSVマッピング・バリデーション
  ↓
馬ごとの評価用データ生成
  ↓
AI指数計算 / 神穴指数計算 / 危険人気馬指数計算
  ↓
ランキング自動更新
  ↓
三連単生成 / WIN5生成 / 結果検証 / 自己進化AI
```

## 既存機能を壊さないための注意点

- 既存のランキング表示データがある場合は、計算結果を上書きする前にバックアップできるようにします。
- 自動計算が失敗した場合でも、手入力の指数・メモを残せるようにします。
- CSV未取込時は、v1.0と同じ手入力運用を継続できるようにします。
- Phase2-2の初期実装では、計算ロジックを既存画面へ一気に統合せず、個別関数と出力確認から段階的に進めます。
