# 橋本競馬AI Phase2-1 実データ連携設計

## 目的

netkeibaやJRAの出馬表・オッズ・結果データを、将来的に橋本競馬AIへ取り込めるように、取込対象・取込方法・CSV標準フォーマット・連動先・実装順を整理します。

Phase2-1では、既存のブラウザ運用、CSV取込、localStorage保存、AI指数・神穴・危険人気馬・三連単・WIN5・結果検証・自己進化ログの導線を壊さず、まずは手動CSVや貼り付けテキストから安全に実データへ接続できる土台を作ります。

## 1. 取込対象データ

橋本競馬AIで予想生成・買い目生成・結果検証・自己進化ログに使うため、以下のデータを取込対象として標準化します。

| データ | 用途 | 備考 |
| --- | --- | --- |
| 出馬表 | レース別の出走馬一覧、馬番、馬名、騎手、調教師などの基本情報 | AI指数計算と買い目生成の基礎データ |
| オッズ | 単勝オッズ、将来的には複勝レンジや券種別オッズ | 神穴判定、危険人気馬判定、期待値判断に利用 |
| 人気 | 人気順、過剰人気、人気薄の判定 | 神穴ランキング、危険人気馬ランキングに利用 |
| 馬体重 | 馬体重、増減、当日気配の補正 | リスク評価、状態面の補正に利用 |
| 調教評価 | 調教ランク、追い切り評価、メモ | AI指数、神穴指数、リスク補正に利用 |
| 結果 | 着順、勝ち馬、入線順 | 結果検証、バックテストに利用 |
| 払戻 | 三連単などの払戻金 | ROI、回収率、万馬券DBに利用 |
| 通過順 | コーナー通過順、特に4角位置 | 展開AI、脚質評価、次走メモに利用 |
| 上がり3F | 末脚性能、レース後評価 | AI指数、展開AI、自己進化ログに利用 |
| 騎手 | 騎手名、騎手補正、乗替情報 | 騎手AI、AI指数補正に利用 |
| 調教師 | 調教師名、厩舎傾向 | 調教師AI、状態・ローテ補正に利用 |
| 血統 | 父、母父、系統、馬場・距離適性 | 血統AI、競馬場OS、神穴条件に利用 |

## 2. 取込方法

Phase2-1では、いきなり自動スクレイピングや外部APIに依存せず、手動運用から将来API連携まで段階的に拡張できる方式にします。

### 手動CSV

- netkeiba、JRA、表計算ソフト、手元メモから橋本競馬AI標準CSVへ整形して取り込みます。
- 最初の標準ルートとして扱い、列名・日付・競馬場・R番号・馬名表記を揃えます。
- 取込前に必須列、数値形式、空欄、重複馬番を確認します。

### 貼り付けテキスト

- Webページやメモからコピーした出馬表・オッズ・結果テキストを貼り付け、将来的にCSVへ変換します。
- Phase2-1では、貼り付け元データを保存ログとして残し、変換ルールの検証材料にします。
- 表記ゆれや余分な空白が発生しやすいため、CSV取込よりもバリデーションを強めます。

### JSONインポート

- `data/*.json` 保存やバックアップ/復元と接続するため、標準CSVと同じ項目をJSONでも扱えるようにします。
- ブラウザのlocalStorageからJSONへ移行する際の中間形式として利用します。
- 予想データ、結果データ、自己進化ログ、バックテスト結果を分離して保存できる構造にします。

### 将来API連携

- 将来的に外部APIや公式データ連携が可能になった場合に備え、入力元と橋本競馬AI内部データを分離します。
- APIレスポンスを直接AI判定に使わず、標準CSV/JSON相当の内部形式へ変換してから利用します。
- データ取得元、取得日時、変換バージョン、取込ログを残し、検証可能性を確保します。

## 3. CSV標準フォーマット

出馬表・予想用データの標準CSVは、1行を「1レース内の1頭」として扱います。

| カラム | 型/例 | 必須 | 説明 |
| --- | --- | --- | --- |
| `date` | `2026-06-02` | 必須 | レース日。YYYY-MM-DD形式を標準とします。 |
| `racecourse` | `東京` | 必須 | 競馬場名。中山、東京、京都、阪神、中京、小倉、新潟、福島など。 |
| `race_number` | `11` | 必須 | レース番号。数値または `11R` から正規化します。 |
| `race_name` | `日本ダービー` | 任意 | レース名。未確定の場合は空欄可。 |
| `distance` | `2400` | 必須 | 距離。メートル数で管理します。 |
| `surface` | `芝` | 必須 | 芝、ダート、障害など。 |
| `going` | `良` | 任意 | 馬場状態。良、稍重、重、不良など。 |
| `horse_number` | `7` | 必須 | 馬番。レース内で重複不可。 |
| `horse_name` | `サンプルホース` | 必須 | 馬名。表記ゆれを後続処理で補正します。 |
| `jockey` | `武豊` | 任意 | 騎手名。騎手AIの補正に利用します。 |
| `trainer` | `サンプル厩舎` | 任意 | 調教師名。調教師AIの補正に利用します。 |
| `popularity` | `3` | 任意 | 人気順。未確定時は空欄可。 |
| `odds` | `8.6` | 任意 | 単勝オッズ。小数で管理します。 |
| `running_style` | `差し` | 任意 | 逃げ、先行、差し、追込など。 |
| `expected_corner4` | `5` | 任意 | 想定4角位置。展開AIと三連単生成に利用します。 |
| `training_grade` | `A` | 任意 | 調教評価。S/A/B/C/Dなどを想定します。 |
| `ai_score` | `82` | 任意 | AI指数。未計算時は空欄可。 |
| `dark_horse_score` | `76` | 任意 | 神穴指数。人気・オッズ・展開利などから算出します。 |
| `risk_score` | `64` | 任意 | 危険人気馬リスク。高いほど危険度が高い想定です。 |

### 標準CSVヘッダー

```csv
date,racecourse,race_number,race_name,distance,surface,going,horse_number,horse_name,jockey,trainer,popularity,odds,running_style,expected_corner4,training_grade,ai_score,dark_horse_score,risk_score
```

### 取込バリデーション方針

- 必須列が存在しないCSVは取り込まず、エラー内容を表示します。
- `date`、`race_number`、`distance`、`horse_number`、`popularity`、`odds`、各スコアは型チェックします。
- 同一 `date`、`racecourse`、`race_number` 内で `horse_number` が重複する場合は警告または取込停止にします。
- `horse_name` が空欄の行は取込対象外にします。
- 任意列が空欄でも、AI自動計算時に「データ不足」として扱えるようにします。

## 4. 結果CSVフォーマット

結果CSVは、1行を「1レース内の1頭の結果」として扱います。レース識別には、取込時に `date`、`racecourse`、`race_number` などのレースキーと紐付ける前提です。

| カラム | 型/例 | 必須 | 説明 |
| --- | --- | --- | --- |
| `finish_position` | `1` | 必須 | 着順。取消・除外は文字列管理も許容します。 |
| `horse_number` | `7` | 必須 | 馬番。出馬表CSVの馬番と突合します。 |
| `horse_name` | `サンプルホース` | 必須 | 馬名。馬番と併用して突合精度を上げます。 |
| `jockey` | `武豊` | 任意 | 実際に騎乗した騎手。乗替確認に利用します。 |
| `popularity` | `3` | 任意 | 確定人気。予想時人気との差分検証に利用します。 |
| `odds` | `8.6` | 任意 | 確定単勝オッズ。期待値検証に利用します。 |
| `corner4_position` | `5` | 任意 | 4角通過順。想定展開とのズレを検証します。 |
| `last_3f` | `34.8` | 任意 | 上がり3F。末脚評価や次走メモに利用します。 |
| `payout_trifecta` | `128600` | 任意 | 三連単払戻。レース単位の払戻として同一値を保持します。 |
| `pace` | `ハイ` | 任意 | レースペース。スロー、平均、ハイなど。 |
| `memo` | `4角外を回して伸びる` | 任意 | 検証メモ、次走注目、敗因など。 |

### 結果CSVヘッダー

```csv
finish_position,horse_number,horse_name,jockey,popularity,odds,corner4_position,last_3f,payout_trifecta,pace,memo
```

### 結果取込時の突合方針

- `horse_number` と `horse_name` を優先して、取込済み出馬表データへ紐付けます。
- 馬番一致・馬名不一致の場合は、表記ゆれ候補として警告します。
- 確定人気・確定オッズは、予想時の人気・オッズと分けて保存できるようにします。
- `corner4_position` と `last_3f` は、展開AI・競馬場OS・自己進化ログの改善材料として扱います。

## 5. 取込後の連動先

取込データは、単に保存するだけでなく、以下の既存機能・Phase2機能へ連動させます。

| 連動先 | 利用データ | 連動内容 |
| --- | --- | --- |
| AI指数ランキング | 出馬表、騎手、調教師、血統、調教評価、馬場、距離 | 各馬のAI指数を表示・再計算する |
| 神穴ランキング | 人気、オッズ、脚質、想定4角、調教評価、血統 | 人気薄かつ期待値がある馬を抽出する |
| 危険人気馬ランキング | 人気、オッズ、馬場、距離、脚質、リスクスコア | 過剰人気や展開不利の人気馬を警告する |
| 三連単生成 | AI指数、神穴指数、危険人気馬、想定展開 | 本線、攻撃型、万馬券型の買い目へ反映する |
| WIN5生成 | 各対象レースのAI指数、波乱度、神穴候補 | 堅軸、準堅軸、波乱、超波乱分類に利用する |
| 結果検証 | 結果、払戻、通過順、上がり3F、確定人気・オッズ | 的中、不的中、回収率、予想ズレを検証する |
| 自己進化ログ | 結果検証メモ、予想ズレ、好走/凡走条件 | 改善提案、採用/保留/却下、次回反映メモに残す |
| バックテスト | 過去の出馬表、予想スコア、結果、払戻 | AI指数・神穴・危険人気馬条件の精度を検証する |

## 6. Phase2-1の実装順

Phase2-1は、ドキュメントとデータ定義を先に固め、既存機能を壊さない形で段階実装します。

### Step1 CSVテンプレート整備

- 出馬表・予想用CSVテンプレートを作成します。
- 結果CSVテンプレートを作成します。
- サンプルCSVと列説明を用意し、手動整形時の迷いを減らします。

### Step2 CSV取込バリデーション

- 必須列チェック、型チェック、空欄チェック、馬番重複チェックを追加します。
- 取込前プレビューで、警告・エラー・取込可能行を確認できるようにします。
- エラー内容は保存ログまたはインポートログに残せるようにします。

### Step3 結果CSV取込

- 結果CSVを既存の出馬表・予想データへ紐付けます。
- 着順、4角位置、上がり3F、払戻、ペース、メモを保存します。
- 確定人気・確定オッズと予想時データの差分を検証できるようにします。

### Step4 保存ログ連携

- 取込日時、取込元、ファイル名、件数、警告、エラーを保存します。
- localStorage、JSONバックアップ、将来のGitHub保存で同じログ構造を扱えるようにします。
- 取込失敗時も、原因を後から確認できるようにします。

### Step5 AI指数自動計算へ接続

- 取込済みの出馬表、人気、オッズ、調教評価、騎手、調教師、血統をAI指数計算へ渡します。
- 神穴指数、危険人気馬リスク、三連単生成、WIN5生成へ連動します。
- 結果検証後は、自己進化ログとバックテストへ接続します。

## 運用上の注意

- netkeibaやJRAなど外部サイトのデータ利用は、各サイトの利用規約・権利・取得方法を確認し、Phase2-1では手動CSV・手動貼り付けを基本にします。
- 自動取得やAPI連携は、利用可能な正規手段が確認できてから実装します。
- 取込データは、予想時データと結果確定後データを分けて保存し、後から検証できる状態を維持します。
- 既存のv1.0機能を壊さないため、最初はドキュメント、テンプレート、バリデーション、ログ保存の順に進めます。
