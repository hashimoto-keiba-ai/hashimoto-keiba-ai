# 橋本競馬AI Phase5-4 実戦テスト運用ガイド

## 目的

Phase5-4の実戦テストでは、実際のレースを1件だけCSVに入力し、橋本競馬AIが本番運用フローとして使えるかを確認します。

確認対象は、CSV取込、AI指数自動計算、危険人気馬、神穴、三連単、WIN5、EV、資金配分、結果検証、自己進化ログ保存、バックアップ保存までの一連の流れです。

## 使用ファイル

- `sample_production_race_template.csv`
  - 実戦テスト用のCSVテンプレートです。
  - 1行につき1頭を入力します。
  - 1レース分の全出走馬を同じ開催日、競馬場、レース番号、レース名で入力します。
- `README.md`
  - 実戦テストの簡易手順を確認できます。
- 本ガイド
  - 本番テストの詳細な流れ、確認観点、保存すべきログを確認できます。

## 事前準備

1. 橋本競馬AIの画面を開きます。
2. 本番運用モードまたはテストモードで操作する前提にします。
3. netkeiba、JRA、競馬新聞など、出馬表、オッズ、馬場状態、騎手、調教師、脚質を確認できる情報源を用意します。
4. `sample_production_race_template.csv` をコピーして、テスト対象レース用のCSVを作成します。
   - 例: `2026-06-07_tokyo_11R_production_test.csv`
5. 1レース分の全出走馬をCSVへ入力します。

## CSV入力ルール

| カラム | 入力内容 | 補足 |
| --- | --- | --- |
| 開催日 | レース開催日 | `YYYY-MM-DD` 推奨 |
| 競馬場 | 競馬場名 | 例: 東京、京都、中山 |
| レース番号 | レース番号 | 例: 11R |
| レース名 | レース名 | 例: 安田記念 |
| 発走時刻 | 発走時刻 | 例: 15:40 |
| 距離 | 距離 | 例: 1600 |
| 芝ダート | 芝またはダート | 例: 芝 |
| 馬場状態 | 良、稍重、重、不良など | 発表後に更新 |
| 頭数 | 出走頭数 | 全行で同じ値 |
| 馬番 | 馬番 | 1頭ごとに入力 |
| 馬名 | 馬名 | 正式馬名推奨 |
| 騎手 | 騎手名 | 乗り替わりに注意 |
| 調教師 | 調教師名 | 分かる範囲で入力 |
| 人気 | 現時点の人気 | 直前オッズで更新推奨 |
| オッズ | 単勝オッズ | 数値で入力 |
| 脚質 | 逃げ、先行、差し、追込など | 予想脚質で可 |
| 想定4角位置 | 4角通過の想定位置 | 例: 1、3、8、12 |
| 調教評価 | A、B、Cなど | 独自評価で可 |
| 馬場適性 | 高、中、低など | 馬場への適性メモ |
| 距離適性 | 高、中、低など | 距離への適性メモ |
| メモ | 補足 | 不安材料、買い材料、直前気配など |

## 実戦テスト手順

### 1. netkeiba等を見ながらCSVに1レース分入力する

- 最初のテストでは、出走頭数が多すぎない平場または特別戦を推奨します。
- 重賞で試す場合は、オッズ変動や直前情報が大きいため、CSV更新時刻をメモに残します。
- netkeiba、JRA、競馬新聞などで出馬表、オッズ、馬場状態、騎手、調教師、脚質を確認します。
- `sample_production_race_template.csv` をコピーして編集します。
- 1頭につき1行入力します。
- 開催日、競馬場、レース番号、レース名、距離、芝ダート、馬場状態、頭数は全行で統一します。
- 人気とオッズは、可能なら発走30分前以降の値に更新します。

### 2. CSV取込を実行する

- 橋本競馬AIのCSV取込画面で、作成したCSVを選択します。
- バリデーション結果を確認します。
- 必須カラム不足、馬番・人気・オッズの数値エラーが出た場合はCSVを修正して再取込します。

### 3. AI指数自動計算を確認する

- 取込後にAI指数ランキングが更新されることを確認します。
- AI指数上位馬、指数差、明らかな異常値がないかを確認します。
- 異常値がある場合は、人気、オッズ、脚質、適性、調教評価の入力ミスを疑います。

### 4. 危険人気馬を確認する

- 危険人気馬ランキングを確認します。
- 人気上位馬のうち、危険判定が出ている馬をメモします。
- 危険理由が、オッズ、脚質、馬場適性、距離適性、展開不利と整合しているか確認します。

### 5. 神穴を確認する

- 神穴ランキングまたは爆穴候補を確認します。
- 人気薄で指数が高い馬を確認します。
- 神穴候補が展開、馬場、距離、騎手、調教のどれで評価されているかをメモします。

### 6. 三連単生成を確認する

- 三連単候補が生成されることを確認します。
- 1着固定、2着候補、3着候補のバランスを確認します。
- 危険人気馬を過度に厚く買っていないか、神穴を3着候補などに含められているかを確認します。

### 7. WIN5候補を確認する

- 対象レースがWIN5対象でない場合でも、候補生成ロジックの表示・保存が崩れないかを確認します。
- WIN5対象レースの場合は、本命候補、相手候補、穴候補の分類を確認します。

### 8. EVを確認する

- EVランキングまたはEV監視欄を確認します。
- 買うべきレースか、見送るべきレースかを判断します。
- オッズ更新前後でEVが大きく変わる場合は、更新時刻を記録します。

### 9. 資金配分を確認する

- 推奨投資額、買い目ごとの配分、総投資額を確認します。
- テストでは実際に購入しない場合でも、仮想投資額を決めてログに残します。
- 資金配分が過大な場合は、見送りまたは投資額上限をメモします。

### 10. レース結果を入力する

- レース確定後、1着、2着、3着、三連単配当、投資額、払戻額を入力します。
- 実ペース、4角位置、馬場傾向、敗因・好走理由も分かる範囲で入力します。

### 11. 自己進化ログを保存する

- 結果検証後、自己進化ログが保存されることを確認します。
- 次回改善ポイントを必ず残します。
- 例: 「稍重の差し有利を過小評価」「逃げ馬の危険人気判定は正解」「オッズ急落馬のEV補正が必要」

### 12. バックアップを保存する

- バックアップ／復元画面で本番運用バックアップをJSON出力します。
- CSV、予想ログ、結果検証、自己進化ログ、資金曲線を同じレース名または開催日で紐づけて保管します。
- 保存したバックアップファイル名と保存場所を推奨ログに残します。

## テスト完了判定

以下をすべて確認できれば、1レース分の本番テストは完了です。

- CSVをエラーなく取り込めた。
- AI指数が自動計算された。
- 危険人気馬を確認できた。
- 神穴候補を確認できた。
- 三連単候補が生成された。
- WIN5候補表示に問題がなかった。
- EVを確認できた。
- 資金配分を確認できた。
- 結果を入力できた。
- 自己進化ログが保存された。
- バックアップJSONを保存できた。
- 操作中に既存画面や既存保存データが壊れていない。

## 推奨ログ保存内容

実戦テスト後は、以下を残すことを推奨します。

- テスト対象レース名
- CSV作成時刻
- オッズ確認時刻
- AI指数1位馬
- 危険人気馬
- 神穴候補
- 三連単候補
- WIN5候補
- EV判断
- 推奨投資額
- 実着順
- 払戻額
- 的中/不的中
- 次回改善ポイント
- バックアップファイル名
- バックアップ保存場所

## 注意事項

- CSVは手入力のため、馬番、人気、オッズの入力ミスが最も起きやすいです。
- 発走直前のオッズ変動により、AI指数、EV、資金配分の判断が変わる可能性があります。
- 最初のPhase5-4テストでは、実購入よりも運用フロー確認を優先してください。
- 既存データを壊さないため、必要に応じて本番運用バックアップを作成してからテストしてください。
