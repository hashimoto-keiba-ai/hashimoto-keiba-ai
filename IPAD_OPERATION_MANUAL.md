# 橋本競馬AI Phase2-5 iPad実戦運用マニュアル

## 目的

このマニュアルは、週末の競馬AI運用を **iPadだけで完結** できるように、開催前日から開催終了後までの実戦フローを整理するためのものです。

Phase2-5では、既存のブラウザ版・iPad対応版の機能を壊さず、Safariまたはホーム画面追加によるアプリ風起動を前提に、CSV取込、AI指数確認、買い目生成、結果検証、自己進化ログ保存までを安定して実行できる運用を目指します。

## 1. iPad運用の目的

iPad運用では、PCを使わずに以下の主要作業を実戦中に実行できる状態を目標にします。

- **出馬表確認**
  - 開催日、競馬場、レース番号、距離、馬場、出走馬、騎手、枠順、馬番を確認する。
  - CSV作成前に、馬名やレース条件の表記ゆれがないか確認する。
- **CSV取込**
  - 出馬表CSV、オッズCSV、結果CSVなどをiPadから取り込む。
  - 取込後は、馬名、人気、オッズ、斤量、騎手、枠順、馬番が正しく反映されているか確認する。
- **AI指数確認**
  - AI指数ランキングを確認し、上位馬、軸候補、相手候補を整理する。
  - 競馬場OS、馬場、展開、騎手補正が必要な場合はメモを残す。
- **神穴確認**
  - 神穴ランキングから爆穴候補、押さえ穴候補を確認する。
  - 人気薄でも展開利、馬場利、コース適性がある馬を買い目候補に残す。
- **危険人気馬確認**
  - 人気上位馬の中で過剰人気、展開不利、馬場不一致、距離不安がある馬を確認する。
  - 三連単やWIN5で軽視、相手まで、完全消しの判断材料にする。
- **三連単生成**
  - AI指数上位、神穴候補、危険人気馬判定を組み合わせて三連単を生成する。
  - 本線、攻撃型、万馬券型の点数バランスを確認する。
- **WIN5生成**
  - WIN5対象レースを堅軸、準堅軸、波乱、超波乱に分類する。
  - A固定、B本線、Cゾーン、D爆発候補の構成を確認し、総点数を調整する。
- **結果検証**
  - レース後に着順、払戻、的中/不的中、購入点数、投資額、回収額を入力する。
  - AI指数、神穴、危険人気馬、三連単、WIN5の判断が正しかったか検証する。
- **自己進化ログ保存**
  - 的中要因、不的中要因、改善案、次回反映メモを保存する。
  - 競馬場OS、神穴条件、危険人気馬条件、WIN5構造の更新候補として残す。

## 2. 週末運用フロー

### 開催前日

開催前日は、出馬表データの準備とAI予想の初期確認を行います。

1. **出馬表確認**
   - 開催競馬場、レース番号、距離、馬場、頭数、出走馬を確認する。
   - 取消、除外、馬名表記、騎手変更がないか確認する。
2. **CSV作成**
   - 出馬表情報を橋本競馬AI用CSV形式に整える。
   - 日付、競馬場、レース番号、馬名、騎手、斤量、枠番、馬番、脚質などを確認する。
3. **CSV取込**
   - iPadのSafariまたはホーム画面アプリ風起動からCSV取込を実行する。
   - 取込後に、馬名、レース条件、枠順、騎手、斤量が画面に反映されているか確認する。
4. **AI指数ランキング確認**
   - AI指数上位馬、軸候補、相手候補を確認する。
   - 競馬場OSや距離別OSと大きくズレる馬がいないか確認する。
5. **神穴ランキング確認**
   - 爆穴候補、押さえ穴候補、想定配当を確認する。
   - 展開利、馬場利、枠順利がありそうな人気薄をメモする。
6. **危険人気馬確認**
   - 人気上位馬の危険度を確認する。
   - 過剰人気、距離不安、馬場不一致、展開不利の根拠を整理する。

### 当日午前

当日午前は、馬場とオッズを確認し、前日予想を実戦向けに補正します。

1. **馬場確認**
   - 天候、馬場状態、芝/ダートの傾向、内外の伸び、前残り/差し傾向を確認する。
   - 競馬場OSの補正が必要な場合は、AI指数補正メモを残す。
2. **オッズ確認**
   - 単勝人気、単勝オッズ、複勝レンジ、人気変動を確認する。
   - 神穴候補の妙味が残っているか、危険人気馬が過剰人気になっているか確認する。
3. **AI指数補正**
   - 馬場、オッズ、騎手変更、馬体重、直前気配を見てAI指数を補正する。
   - 補正理由は結果検証で振り返れるようにメモする。
4. **勝負レース選定**
   - AI指数、神穴、危険人気馬の判断が揃っているレースを優先する。
   - 不確定要素が多いレースは見送り候補にする。

### レース直前

レース直前は、買い目と最終結論を保存し、直前変更に備えます。

1. **買い目生成**
   - 三連単自動生成を実行し、本線、攻撃型、万馬券型を確認する。
   - 点数が増えすぎる場合は、期待値や危険人気馬判定を使って絞り込む。
2. **危険人気馬再確認**
   - 直前オッズで人気が集中している馬を再確認する。
   - 危険度が上がった馬は、頭固定を避ける、相手までにする、消すなどの判断を行う。
3. **WIN5候補確認**
   - WIN5対象レースのA/B/C/D候補を確認する。
   - 堅軸レースと波乱レースのバランス、総点数、資金配分を確認する。
4. **最終結論保存**
   - 最終印、買い目、見送り判断、資金配分、補正理由を保存する。
   - レース後の検証で、最終判断と結果を比較できる状態にする。

### レース後

レース後は、結果入力と検証ログ保存を行い、次回改善につなげます。

1. **結果入力**
   - 着順、勝ち馬、2着馬、3着馬、人気、馬場、展開を入力する。
   - 的中/不的中、購入点数、投資額、回収額を記録する。
2. **三連単配当入力**
   - 三連単配当、払戻、的中買い目、外れた場合のズレを入力する。
   - 本線、攻撃型、万馬券型のどの構成が有効だったか確認する。
3. **検証ログ保存**
   - AI指数の評価、神穴候補の走り、危険人気馬の結果を検証する。
   - 的中要因、不的中要因、買い目の過不足を保存する。
4. **OSアップデート**
   - 競馬場OS、距離別OS、馬場バイアス、脚質傾向の更新候補を整理する。
   - 即時反映ではなく、検証済みルールとして採用/保留/却下を判断する。
5. **自己進化ログ保存**
   - 改善提案、次回反映メモ、採用/保留/却下の判断を保存する。
   - AI指数、神穴、危険人気馬、WIN5構造の改善候補として残す。

### 開催終了後

開催終了後は、週末全体の成績を整理し、次回運用へ改善点を引き継ぎます。

1. **回収率入力**
   - 週末の投資額、回収額、的中率、回収率を入力する。
   - 券種別、競馬場別、レースタイプ別に振り返る。
2. **バックテスト**
   - AI指数上位、神穴候補、危険人気馬判定、三連単生成、WIN5生成の結果を検証する。
   - 「買うべきだった馬」「消すべきだった馬」「点数を削るべきだった条件」を整理する。
3. **次回改善点整理**
   - 競馬場OS、馬場バイアスAI、神穴AI、危険人気馬AI、WIN5 AIへの改善候補をまとめる。
   - 次回開催前に確認するチェック項目として保存する。

## 3. iPadホーム画面運用

iPadでは、Safariで開く運用と、ホーム画面へ追加してアプリ風に起動する運用を基本にします。

1. **Safariで開く**
   - GitHub Pages公開URL、またはローカル確認用URLをSafariで開く。
   - 初回表示時は、トップ画面、CSV取込、AI指数、神穴、危険人気馬、三連単、WIN5、結果検証、自己進化ログの導線を確認する。
2. **ホーム画面に追加**
   - Safariの共有ボタンから「ホーム画面に追加」を選択する。
   - ホーム画面上のアイコン名を「橋本競馬AI」など分かりやすい名前にする。
3. **アプリ風起動**
   - ホーム画面のアイコンから起動し、アプリ風に運用する。
   - Safariのタブが増えすぎるのを防ぎ、週末運用時に素早く起動できるようにする。
4. **横画面推奨**
   - AI指数ランキング、買い目、WIN5構成、検証ログは情報量が多いため横画面を推奨する。
   - 入力フォームが見づらい場合は、横画面に切り替えてから入力する。

## 4. トラブル対応

### 画面が崩れた場合

- iPadを横画面に切り替える。
- Safariまたはホーム画面アプリを再読み込みする。
- 拡大表示になっている場合は、表示倍率を戻す。
- それでも崩れる場合は、Safariで直接URLを開き直す。

### データが消えた場合

- まず別タブや別ブラウザで開いていないか確認する。
- localStorage保存の場合、同じURLで開いているか確認する。
- GitHub PagesのURL、ローカルURL、別ブランチURLを混在させない。
- 直近のJSONバックアップがある場合は、復元手順へ進む。

### バックアップ復元

- 復元前に、現在の状態を可能な範囲でバックアップする。
- 保存済みのJSONバックアップから対象データを復元する。
- 復元後は、出馬表、AI指数、買い目、結果検証、自己進化ログが戻っているか確認する。
- 復元した日付、対象競馬場、復元理由を自己進化ログまたは運用メモに残す。

### CSVが読めない場合

- CSVの文字コード、改行コード、区切り文字を確認する。
- ヘッダー行の列名が橋本競馬AI用の項目名と一致しているか確認する。
- 日付、競馬場、レース番号、馬名、騎手、馬番、枠番、人気、オッズなどの必須項目が欠けていないか確認する。
- iPadのファイルアプリ上でCSVが正しく保存されているか確認する。
- 読み込めない場合は、表計算アプリで開き直してCSV形式で再保存する。
