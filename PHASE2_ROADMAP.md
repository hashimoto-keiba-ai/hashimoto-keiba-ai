# 橋本競馬AI Phase2 ロードマップ

## 目的

橋本競馬AI v1.0完成後の次段階として、実データ連携・自動化・AI強化の計画を整理します。

v1.0では、ブラウザ運用、iPad対応、CSV取込、localStorage保存、GitHub Pages公開、AI指数・神穴・危険人気馬・三連単・WIN5・ROI・自己進化ログの基本導線を完成版として管理します。

Phase2では、これらの既存機能を壊さず、以下の方向で段階的に強化します。

- 実データ連携を強化し、手入力依存を減らす
- AI予想エンジンの自動計算と判定精度を高める
- localStorage中心の保存からJSON/GitHub保存へ移行する
- バックテスト結果を自己進化AIへ反映する
- iPadで週末の実戦運用を完結できる流れを整える

## Phase2 全体方針

| 項目 | 方針 |
| --- | --- |
| 開発方針 | v1.0のブラウザ運用版を維持しながら、ドキュメント・データ保存・CSV取込・AI判定を段階的に追加する |
| 安全方針 | 既存の画面、localStorage保存、バックアップ/復元、Pages公開構成を壊さない |
| データ方針 | 手入力、CSV取込、JSON保存、GitHub API保存を同じデータ構造で扱えるようにする |
| AI方針 | AI指数、神穴指数、危険人気馬、三連単、WIN5、自己進化ログを連動させる |
| 運用方針 | PCで設計・整理し、iPadで週末のCSV取込、予想生成、結果検証まで行える状態を目指す |

## Phase2-1 実データ連携

### 目的

出馬表、オッズ、レース条件、結果情報などの実データを扱いやすくし、手入力だけに依存しない運用へ移行します。

詳細な取込対象、取込方法、CSV標準フォーマット、結果CSVフォーマット、連動先、実装順は [`DATA_IMPORT_DESIGN.md`](DATA_IMPORT_DESIGN.md) に整理します。

### 実施項目

- **CSV取込強化**
  - 取込項目のマッピング機能を整備する
  - 馬名、騎手、斤量、人気、オッズ、枠番、馬番、脚質、距離、馬場などの項目を標準化する
  - 不足項目や形式違いを検出するバリデーションを追加する
  - 取込前プレビューと取込後ログを残す
- **netkeiba手動CSV変換**
  - netkeiba等から手動で取得した出馬表情報を、橋本競馬AI用CSVへ変換する手順を整理する
  - コピー&ペースト、表計算ソフト、CSV整形の運用ルールを作る
  - 変換後の列名、日付、競馬場、R番号、馬名表記ゆれを統一する
- **JRA出馬表データ取込準備**
  - JRA出馬表データを取り込むための項目定義を準備する
  - 将来的な自動取得や半自動取込に備え、入力元と保存先のデータ構造を分離する
  - JRA公式データ、手動CSV、その他CSVの差分を吸収できる設計にする
- **オッズ手入力/CSV取込**
  - 単勝オッズ、複勝レンジ、人気、直前人気変動を手入力できる形にする
  - オッズCSVがある場合は一括取込できるようにする
  - AI指数、神穴指数、危険人気馬判定でオッズ情報を参照できるようにする

### 完了目安

- 出馬表CSVを取り込み、AI指数計算に必要な基本項目を画面上で確認できる
- オッズ情報を手入力またはCSV取込し、神穴・危険人気馬判定に利用できる
- 取込ログとエラー内容を保存できる

## Phase2-2 AI予想エンジン強化

### 目的

v1.0で用意したAI指数・神穴・危険人気馬・三連単・WIN5の画面とデータ構造を、より自動化された予想エンジンへ発展させます。

AI指数・神穴指数・危険人気馬指数の構成要素、基本計算式、競馬場別補正、距離別補正、実装順の詳細は [`AI_SCORE_ENGINE_DESIGN.md`](AI_SCORE_ENGINE_DESIGN.md) に整理します。

### 実施項目

- **AI指数自動計算**
  - 基礎能力、近走成績、コース適性、距離適性、馬場適性、騎手補正、調教師補正を点数化する
  - 競馬場OSの補正を指数へ反映する
  - 入力データ不足時は暫定指数として扱い、根拠を明示する
- **神穴指数自動計算**
  - 人気薄、オッズ妙味、展開利、馬場バイアス、枠順、脚質、騎手/調教師の穴傾向を点数化する
  - 万馬券DBと神レースDBの再現条件を参照する
  - 爆穴候補と押さえ穴候補を分ける
- **危険人気馬自動判定**
  - 過剰人気、展開不利、馬場ミスマッチ、距離不安、ローテ不安、枠順不利を自動評価する
  - 人気上位馬の中から危険度をランク化する
  - 判定根拠を買い目生成時に反映する
- **三連単自動生成精度向上**
  - AI指数上位、神穴候補、危険人気馬除外/軽視を組み合わせる
  - 本線、攻撃型、万馬券型の比率をレースタイプごとに変える
  - 点数が増えすぎる場合は期待値順に絞り込む
- **WIN5構造自動判定**
  - 各対象レースを堅軸、準堅軸、波乱、超波乱に分類する
  - A固定、B本線、Cゾーン、D爆発候補の配分を自動提案する
  - 全体点数と期待値のバランスを調整する

### 完了目安

- CSV/手入力データからAI指数・神穴指数・危険人気馬判定を自動生成できる
- 三連単とWIN5の買い目に、自動計算結果が反映される
- 各判定に「なぜその評価になったか」の根拠メモが残る

## Phase2-3 GitHub保存強化

### 目的

localStorage中心の保存から、JSONファイル保存とGitHub API保存へ段階的に移行し、端末依存を減らします。

保存対象JSON、保存方式、GitHub API設計、保存前安全処理、設定項目、実装順の詳細は [`GITHUB_JSON_STORAGE_DESIGN.md`](GITHUB_JSON_STORAGE_DESIGN.md) に整理します。

### 実施項目

- **localStorageからJSON保存へ移行**
  - localStorageを一時保存、JSONを正本保存として扱う設計へ移行する
  - 既存localStorageデータをJSON形式でエクスポートできるようにする
  - JSONからlocalStorageへ復元する互換性は維持する
- **data/*.json への保存**
  - 出馬表、馬データ、予想、結果、OS更新、WIN5、ROI、買い目、バックアップを `data/*.json` に保存する
  - データ種別ごとに保存ファイルを分ける
  - JSONスキーマまたは項目定義を整備する
- **バックアップ自動化**
  - 予想前、結果入力後、自己進化ログ更新後にバックアップを作る流れを整える
  - 日付・競馬場・操作種別が分かるバックアップ名にする
  - 復元前に現状バックアップを作成する安全策を入れる
- **GitHub API保存の本実装**
  - GitHub Contents APIを使った読込/保存を実装する
  - SHA更新、競合検出、保存失敗時のエラー表示に対応する
  - GitHub Tokenの扱い、保存対象ブランチ、コミットメッセージを整理する

### 完了目安

- `data/*.json` を標準保存先として扱える
- GitHub API経由で保存・読込・更新できる
- localStorage、JSONバックアップ、GitHub保存の役割が明確になる

## Phase2-4 自己進化AI強化

### 目的

結果検証とバックテストをもとに、AIの重み・条件・競馬場OSを継続的に改善する仕組みを強化します。

自己進化AIの目的、入力データ、改善対象AI、改善ロジック、自己進化ログ形式、実装順の詳細は [`SELF_EVOLUTION_AI_DESIGN.md`](SELF_EVOLUTION_AI_DESIGN.md) に整理します。

### 実施項目

- **バックテスト結果から重み調整**
  - 的中、不的中、回収率、期待値、危険人気馬の飛び率を集計する
  - AI指数、神穴指数、危険人気馬判定の重みを見直す
  - 採用/保留/却下の改善ログを重み変更候補として扱う
- **競馬場別OS自動更新**
  - 競馬場、距離、馬場、脚質、枠順ごとのズレを検出する
  - 中山、東京、京都、阪神、中京、小倉、新潟、福島などの競馬場OSへ更新案を保存する
  - 更新案は即時反映ではなく、検証済みルールとして段階採用する
- **危険人気馬条件の自動更新**
  - 危険人気馬が実際に凡走した条件を蓄積する
  - 逆に危険判定した馬が好走した失敗条件も保存する
  - 過剰人気、距離、馬場、展開、ローテ、騎手乗替の条件を再評価する
- **神穴条件の自動更新**
  - 神穴候補が好走した条件を蓄積する
  - 神穴候補が不発だった条件を整理する
  - 万馬券DB、神レースDB、馬場バイアスAIへ反映する

### 完了目安

- 結果検証ログから改善候補を自動抽出できる
- 競馬場OS、危険人気馬条件、神穴条件の更新案を保存できる
- 自己進化ログが次回予想の補正候補として利用される

## Phase2-5 iPad実戦運用

### 目的

週末の実戦で、iPadだけでもデータ取込、AI指数確認、買い目生成、結果検証、自己進化ログ保存まで行える運用フローを整えます。

### 実施項目

- **iPadホーム画面運用**
  - GitHub Pages公開URLをSafariからホーム画面へ追加して運用する
  - 画面遷移、入力フォーム、CSV取込、バックアップ操作をiPadで確認する
  - タッチ操作と1カラム表示で見やすい構成を維持する
- **CSV取込→AI指数→買い目生成**
  - 出馬表CSVまたは手動変換CSVをiPadで取り込む
  - AI指数、神穴指数、危険人気馬を生成する
  - 三連単、WIN5の買い目を自動生成し、点数と根拠を確認する
- **結果検証→自己進化ログ保存**
  - レース後に着順、払戻、的中/不的中、ROI、反省点を入力する
  - 改善提案を自己進化ログへ保存する
  - 次回反映メモを競馬場OSやAI研究所の更新候補として残す
- **週末運用フロー**
  - 金曜/前日: 出馬表CSV準備、競馬場OS確認、注目レース整理
  - 当日午前: オッズ入力、AI指数更新、危険人気馬チェック
  - 発走前: 三連単/WIN5買い目生成、点数調整、バックアップ
  - レース後: 結果入力、ROI確認、自己進化ログ保存
  - 日曜夜/週明け: 週末総括、競馬場OS更新案、Phase2改善タスク整理

### 完了目安

- iPadのホーム画面から橋本競馬AIを起動し、実戦運用の主要操作を完結できる
- CSV取込から買い目生成までの流れが明文化される
- 結果検証と自己進化ログが週末運用に組み込まれる

## Phase2 優先順位

| 優先度 | 項目 | 理由 |
| --- | --- | --- |
| 1 | CSV取込強化 | 実データ連携とAI自動計算の土台になるため |
| 2 | data/*.json保存 | localStorage依存を減らし、GitHub保存へ進むため |
| 3 | AI指数/神穴/危険人気馬の自動計算 | 買い目生成精度に直接影響するため |
| 4 | GitHub API保存 | 複数端末運用と永続保存を実現するため |
| 5 | 自己進化AI強化 | 長期的な精度改善と競馬場OS更新に必要なため |
| 6 | iPad週末運用フロー | 実戦で使いやすい運用手順として定着させるため |

- 三連単とWIN5の自動生成に、強化後のAI判定が反映される
- `data/*.json` とGitHub API保存で運用データを保持できる
